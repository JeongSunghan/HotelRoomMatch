## 1. 디자인 컨셉: "Digital Guest List (디지털 게스트 리스트)"
호텔 측의 상세 도면(엘리베이터 위치, 뷰 등) 부재라는 제약을 역이용하여, 물리적 지도 대신 **추상적이고 직관적인 '카드 그리드' 시스템**을 도입합니다.
영화관 좌석 예매 시스템의 **'설렘'**과 게임 파티 매칭의 **'직관성'**을 결합하여, 단순 행정 절차가 아닌 **네트워킹의 시작점**으로 경험을 재설계합니다.

---

## 2. 주요 UI/UX 개선 포인트

### A. 레이아웃 (Layout): 벤토 그리드 (Bento Grid) 재해석
* **AS-IS:** 단순 나열식 리스트 (스크롤이 길고 정보 파악이 힘듦)
* **TO-BE:** **밀도 높은 카드 그리드 시스템**
    * 화면 중앙에 방 카드들을 촘촘하게 배열 (CSS Grid 활용).
    * 단순 나열을 피하고, **10개 단위** 혹은 **복도(홀수/짝수)** 기준으로 미세한 간격(Gap)을 주어 시각적 그룹핑 유도.
    * 스크롤을 최소화하여 한눈에 전체 현황을 파악(Dashboard View) 가능하도록 구성.

### B. 방 카드 디자인 (Component): 아바타 슬롯 시스템
* **AS-IS:** 텍스트 중심 (`0/2`, `빈 방`)
* **TO-BE:** **시각적 슬롯 & 아바타 (Avatar Slots)**
    * **슬롯 메타포:** 방 카드 안에 동그라미 슬롯 2개(2인실 기준) 배치.
    * **빈 방:** 점선으로 된 빈 원 (Empty State).
    * **1인 입장:** 프로필 이미지(또는 이니셜 아이콘) 1개 + 빈 원 1개.
    * **만실:** 꽉 찬 원 2개 + 카드 전체가 'Sold Out' 느낌으로 딤드(Dimmed) 처리.
    * **효과:** 텍스트를 읽지 않아도 '빈자리'를 본능적으로 인지 가능.

### C. 인터랙션 (Interaction): 마이크로 인터랙션 & 티켓팅
* **AS-IS:** 라디오 버튼 선택 → 등록 버튼 클릭 (정적임)
* **TO-BE:** **즉각적인 피드백과 몰입감**
    * **Hover:** 마우스 오버 시 카드가 살짝 떠오르거나(Scale Up), 테두리에 브랜드 컬러 Glow(발광) 효과.
    * **Active (선택):** 클릭 시 영화관 좌석처럼 색상이 반전되거나 체크 표시 애니메이션 작동.
    * **Confirm:** 방 선택 완료 시, 하단에서 **'티켓' 형태의 요약 카드**가 슬라이드 업(Slide-up) 되며 최종 확인 유도.

## 3. 화면 구성 제안 (Wireframe Description)

### [Header Area] : 내 상태 대시보드
* 기존의 긴 통계 바를 **벤토 그리드 형태의 컴팩트한 모듈**로 변경.
* **좌측 모듈:** [ 나의 방 티켓 ] (미배정 시 '좌석을 선택해주세요' / 배정 시 '705호 | 트윈' 티켓 활성화)
* **우측 모듈:** [ 잔여 현황 ] (남/여 잔여석을 심플한 도넛 차트나 숫자로 표시)

### [Main Content] : 인터랙티브 룸 맵 (Room Map)
* **필터링 탭:** `전체 보기` | `빈 방만 보기` | `내 성별만 보기` (토글 형태)
* **그리드 영역:**
    ```text
    [ 701 ]       [ 702 ]       [ 703 ]
    (👤)(◌)       (◌)(◌)       (👤)(👤)
     트윈          트윈           트윈
    ```
    * 각 카드는 클릭 가능한 버튼 역할.
    * 나와 다른 성별의 방은 흐리게 처리(Opacity 50%)하거나 빗금 패턴으로 비활성 표시.

### [Footer/Modal] : 액션 바
* 방을 선택하면 하단에 고정된(Sticky) 바(Bar)가 나타나며, **[ 701호 선택 완료 ]** 버튼 활성화.

## 4. 디자인 스타일 (Look & Feel)

* **Color Palette:**
    * **배경:** 짙은 네이비(Dark Navy) 또는 차콜(Charcoal)로 고급스러운 라운지 분위기 연출 (Dark Mode 권장).
    * **포인트:** 형광 라임(Lime), 앰버(Amber), 또는 브랜드 컬러(Vivid Blue)를 사용하여 어두운 배경에서 **상태값(빈 방, 내 방)**이 네온사인처럼 돋보이게 함.
* **Typography:** 가독성이 좋은 산세리프(San-serif) 폰트 (Pretendard 등), 숫자는 굵게(Bold) 처리하여 호수 식별성 강화.

## 5. 개발 가이드 (Implementation Tips)

* **CSS Grid 활용 (반응형):**
    ```css
    .room-grid {
      display: grid;
      grid-template-columns: repeat(auto-fill, minmax(140px, 1fr)); /* 카드 최소 너비 설정 */
      gap: 16px;
    }
    ```
* **상태 관리 (State):**
    * 각 방 객체(Object)에 `occupants: []` 배열을 두어, 배열 길이에 따라 UI(슬롯 채움)를 렌더링.
    * `isLocked`, `gender` 속성을 통해 비활성 스타일(Disabled Class) 적용.
