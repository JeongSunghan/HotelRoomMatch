## 💡 핵심 교훈

### 1. Mock은 실제 API와 동일하게
```typescript
// ❌ Bad
getDocs: vi.fn()

// ✅ Good
getDocs: vi.fn(() => Promise.resolve({
    empty: true,
    size: 0,
    docs: [],
    forEach: vi.fn(),
}))
```

### 2. UI 테스트는 data-testid 사용
```typescript
// ❌ Bad - 동적 텍스트
expect(screen.getByText('10')).toBeInTheDocument();

// ✅ Good - 안정적인 ID
expect(screen.getByTestId('male-available')).toHaveTextContent('10');
```

### 3. 텍스트 중복 시 정규식 활용
```typescript
// ❌ Bad - "101"의 0과 충돌
expect(screen.getByText(/0/)).toBeInTheDocument();

// ✅ Good - 정확한 패턴
expect(screen.getByText(/0\s*\/\s*2/)).toBeInTheDocument();
```

### 4. 복잡한 Unit Test < Integration Test
- userStays의 복잡한 Mock보다 Integration test가 더 효과적
- 실제 동작을 더 잘 검증하고 유지보수가 쉬움
