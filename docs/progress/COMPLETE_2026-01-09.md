## 💡 핵심 교훈

### 1. Mock 완전성이 중요
Firebase Mock은 실제 API와 동일한 구조로 작성해야 함
```typescript
// ❌ Bad
getDocs: vi.fn()

// ✅ Good
getDocs: vi.fn(() => Promise.resolve({
    empty: true,
    size: 0,
    docs: [],
    forEach: vi.fn(),
}))
```

### 2. UI 테스트는 data-testid 사용
텍스트나 레이아웃 변경에 강건한 테스트 작성
```typescript
// ❌ Bad
expect(screen.getByText('10')).toBeInTheDocument();

// ✅ Good
<span data-testid="male-available">{value}</span>
expect(screen.getByTestId('male-available')).toHaveTextContent('10');
```

### 3. 신규 테스트는 실제 컴포넌트 확인 후 작성
추측으로 작성하지 말고, 실제 렌더링 결과를 보고 작성
```typescript
// 1. 먼저 컴포넌트 렌더링
render(<Component />);
screen.debug(); // 실제 HTML 구조 확인

// 2. 확인된 구조로 테스트 작성
expect(screen.getByRole('button')).toBeInTheDocument();
```
